# Runbook

Run commands from the `architecture_planning/` directory.

## Requirements

| Dependency | Version |
|---|---|
| Python | 3.11+ |
| Ollama | latest |
| Model | `llama3.1` |
| Docker/Compose | optional |

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `OLLAMA_HOST` | `http://127.0.0.1:11434` | Ollama endpoint |
| `OLLAMA_MODEL` | `llama3.1` | Model for both agents |

## Local installation

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e .
```

## Usage and operation

### Ingest business requirements

```bash
python architecture_planner.py \
  --input "Need a highly available medical billing platform with multi-region failover" \
  --thread-id design-001 \
  --max-rounds 3
```

### Extract the final architecture document

```bash
python architecture_planner.py \
  --input "Need a highly available medical billing platform with multi-region failover" \
  --output /tmp/final_architecture.md \
  --show-history
```

`--output` writes the unified Markdown master specification generated by the debate loop.

## Docker usage

```bash
docker compose up -d ollama
docker compose exec ollama ollama pull llama3.1
docker compose run --rm architecture_planner \
  --input "Need a resilient distributed platform for digital claims processing" \
  --output /tmp/final_architecture.md
```
